# Decisions Log

- **S1 — Foundation:** Next.js pinned to 15.x per §2 (create-next-app now defaults to 16). shadcn-style UI primitives hand-rolled on Radix (popover) and themed to §15 tokens; identical API, no extra component library. `.env.example` added.
- **S2 — Database:** Migrations 0001–0008 cover the full blueprint §38 schema. `question_bank`, `personas`, `rubric_prompts`, and `usage_log` have **no user read policy** — answers/prompts are served only by server code via the service client. Types generated from a live Postgres into `src/types/database.ts` (Supabase CLI gen-types needs Docker, unavailable in the build environment; the script output matches the CLI format). Seed uses fixed UUIDs so deep links are deterministic. 7 "In Training" stubs seeded per §5 (Secular Skeptic deferred — §5 says 7 stubs); stub one-liners live in code since `tracks` has no description column (schema is locked).
- **S3 — Auth + shell:** Profile + name created by a DB trigger on signup (`handle_new_user`). Middleware guards all app routes; logged-in users are bounced from `/`, `/login`, `/signup` to the dashboard.
- **S4 — Dashboard + tracks:** "Lesson read" = `xp_events` row with reason `lesson_complete`; "lesson complete" = a passed checkpoint attempt — no separate lesson_progress table needed for v1.
- **S5 — Lesson reader:** Lesson copy ships verbatim; glossary callouts are listed in `content.terms` (rendered as tap-open chips) so the approved prose is untouched. Added `POST /api/learn/[lessonId]/complete` (not in §4's list but required by the LessonCompleteCTA — simplest implementation). Verses render client-side through `/api/verses` so the ESV cache write-through is exercised on every view.
- **S6 — Quiz:** Grading fully server-side; the response reveals correctness + teaching note + source-block deep link but never the correct option. Pass threshold read from `quizzes.config` (0.7). XP on first pass only.
- **S7 — Flashcards:** Interval snapping interpreted as nearest-of-{1,3,7,14,30,90} until raw > 90 (matches the acceptance sequence 1 → 3 → 7). Scripture mode ladder advances when the new interval reaches ≥7; `again` regresses one mode step. Queue-completion XP capped at once per day.
- **S8 — Respond:** Drill objections are the lesson's steelmanned pushbacks, kept in code (`lib/ai/prompts/respond.ts`) and rotated daily — no schema change. Scoring via Sonnet (`claude-sonnet-4-6`) with Zod-validated output; every call logged to `usage_log`.
- **S9 — Simulator:** Persona turns via Haiku (`claude-haiku-4-5`), temperature 0.8, streamed as plain text (AI SDK `streamText` server-side; a lightweight reader client-side instead of `useChat` — fewer moving parts, same streamed UX). Messages persisted server-side on each turn; refresh restores the thread from the DB.
- **S10 — Coach:** Citation-or-silence enforced in code: invalid `citation_source_id` → one retry → nulled + `flagged=true`. Quoted moments are checked against the transcript; misses flag the review for audit. Simulation XP scales 50–250 on the rubric average. Focus mode (`data-mode="focus"`) wraps /review, /respond, and all /spar pages.
- **v0.2 — Feedback round 1 (Melo):** Honest scoring: rubric prompt v2 (DB) + scoring-discipline rules in both AI prompts; a non-answer scores 1s, and respond/spar XP now scales from 0 (avg 1 → 0 XP). Profile icon opens a menu (account/how-it-works/sign out) instead of signing out; new /account page with stats, name editing, and plain-English explanations of XP/streak/levels. Review session got an instructions screen and the grade buttons got plain-language hints. Inline scripture references in prose are auto-detected and open verse popovers (/api/verses made public + CDN-cached for the marketing pages). Glossary terms got full multi-sentence definitions (DB). Trinity lesson deepened from 16 to 24 blocks (echad + homoousios word studies, wider deity case, Nicaea history, harder follow-ups, why-it-matters) and the checkpoint went from 8 to 24 questions — all in update-app-v02.sql since content lives in the DB. New public site: /about (philosophy + What Contend Affirms), /training catalog with a preview article per subject (copy in src/lib/site-content.ts), /how-it-works (method, scoring, XP/streak/levels), shared site header/footer, richer landing. Course model reframed: finish any course to unlock the next choice, completed courses stay open (display logic; gate enforcement still a later milestone). Design pass: scripture/word-study set pieces, drop caps, dashboard 4-card layout, paper grain, hover lifts. Google button now surfaces provider-not-enabled errors; enabling Google OAuth requires Supabase + Google Cloud setup (user-side).
